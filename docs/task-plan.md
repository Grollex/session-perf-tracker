### Шаг 1: Анализ текущего состояния проекта
- Проверить статус Git репозитория.
- Определить, какие файлы были изменены и не закоммичены.

### Шаг 2: Проверка необходимых инструментов
- Убедиться, что все необходимые инструменты установлены и доступны в PATH.
- Если отсутствуют ffmpeg или ffprobe, установить их.

### Шаг 3: Анализ архитектуры проекта
- Прочитать файлы `ARCHITECTURE.md`, если он существует.
- Понять основные компоненты и зависимости проекта.

### Шаг 4: Определение задачи пользователя
- Уточнить, что именно пользователь хочет сделать или узнать.
- Если задача не четко определена, запросить дополнительную информацию.

### Шаг 5: Разработка плана решения задачи
- Разбить задачу на более мелкие шаги.
- Определить необходимые команды и действия для выполнения каждого шага.

### Шаг 6: Выполнение действий по плану
- Постепенно выполнять каждый шаг, проверяя результаты после каждого действия.
- Если возникают ошибки или неожиданные ситуации, диагностировать и корректировать план.

### Шаг 7: Обновление проектной памяти
- Добавить в `PROJECT_MEMORY.md` только те факты, которые были проверены и подтверждены.
- Записывать изменения и причины их выполнения.

### Шаг 8: Финальная проверка и завершение задачи
- Убедиться, что все шаги выполнены и задача решена.
- Завершить работу с сообщением о завершении.
